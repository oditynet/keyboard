# Теория: 

### Этап 1: Ядро — Получение "сырых" данных
Всё начинается в момент, когда ты нажимаешь клавишу на своей кастомной клавиатуре.

Скан-код (Hardware Scancode): Микроконтроллер отправляет по USB не символ, а структурированный пакет данных. Его формат определяется HID Report Descriptor. Этот пакет содержит информацию о том, какие клавиши-модификаторы (Ctrl, Alt, Shift) нажаты, и список USB HID Usage ID для остальных клавиш. Важно понимать, что это — скан-коды, абстрактные номера, соответствующие физическим клавишам, а не буквы.

```
| MODIFIER | RESERVED |   KEY1   |   KEY2   |   KEY3   |   KEY4   |   KEY5   |   KEY6   |
```
Например, для буквы 'a' отправляется: 

```
[0, 0, 4, 0, 0, 0, 0, 0]
```

Обработка ядром: Ядро Linux получает этот пакет через подсистему USB. Драйверы ядра преобразуют полученные данные в коды клавиш (keycodes). Эти коды уже являются внутренними идентификаторами для каждой клавиши. Посмотреть, что видит ядро от твоей клавиатуры, можно с помощью команды evtest (тебе нужно будет выбрать своё устройство из списка). На этом уровне работа с клавиатурой заканчивается.

### Этап 2: X-сервер и XKB — Магия превращения кода в символ
Дальше эстафету перехватывает X-сервер (или Wayland-композитор) и его мощнейший компонент — X Keyboard Extension (XKB). XKB получает от ядра keycode и, используя сложную систему конфигурационных файлов, превращает его в keysym, то есть в символ клавиши.

Вся логика XKB разбита на несколько уровней:

xkb_keycodes: Здесь keycode (например, 57) связывается с символическим именем, например, <AC01>.

xkb_types: Определяет, как клавиши-модификаторы (Shift, Ctrl, Alt) влияют на значение. Например, для обычной буквы тип TWO_LEVEL означает, что без Shift — одно значение, с Shift — другое.

xkb_symbols: Это самое главное. Здесь для каждого символического имени клавиши (<AC01>) и для каждой группы (group) прописано, какой символ (keysym) выдавать. Группа — это и есть наша "раскладка клавиатуры" (например, английская или русская). Строка key <AC01> { [ a, A ] }; означает, что клавиша в первой группе (первой раскладке) без Shift даст 'a', а с Shift — 'A'. Для второй группы (русской) там будет что-то вроде { [ ф, Ф ] }.

Сами файлы раскладок (символов) хранятся в каталоге /usr/share/X11/xkb/symbols/ (или /etc/X11/xkb/symbols/ в старых системах). Например, за русскую раскладку отвечает файл ru. В нём можно найти описание того, какой английской клавише соответствует какая русская буква.

char keyLayers[3][ROWS][COLS] = {

    {'h', 'j', 'k', 'l', ';', '\''},     // Строка 2

};
